Using the app day-to-day

Starting the app

Each time you want to use the app, open Terminal, navigate to the app folder, and run:

Terminal
bash run.sh

The app will open in your browser at http://localhost:8501.

Stopping the app

When you're finished, go back to the Terminal window and press Ctrl+C. You can then close Terminal.

Getting updates

When a new version is released, download the ZIP from GitHub again, replace the files in your app folder (keep your config.yaml), and run bash setup.sh once to update packages.

Troubleshooting

Terminal says "No such file or directory"
Terminal isn't in the right folder. Make sure you've navigated to the app folder first ‚Äî type cd then drag the app folder into Terminal and press Enter. Then try the command again.
Setup seems to hang or take a very long time
This is normal on the first run ‚Äî some components can take 5‚Äì10 minutes to download on a slow connection. Leave Terminal open and let it finish. You'll see ‚úÖ Setup complete! when it's done.
The app opens but shows "config.yaml not found"
You're running the command from the wrong folder. Navigate to the app folder first: type cd then drag the folder into Terminal and press Enter, then run bash run.sh again.
"No module named streamlit" error
Setup didn't complete successfully. Run bash setup.sh again from inside the app folder. If you see any red error messages, take a screenshot and send it to the person who set up this app.
PDF generates but is blank or looks wrong
The PDF rendering libraries may not have installed correctly. Run bash setup.sh again, then close Terminal completely and reopen it before running bash run.sh.
"Port 8501 already in use" error
The app is already running in another Terminal window. Either use it by going to http://localhost:8501 in your browser, or find the other Terminal window and press Ctrl+C to stop it first.
Setup asks for a password and I'm not sure what to enter
Enter your normal Mac login password ‚Äî the same one you use when you unlock your computer or install an app. This is your own account password and does not require IT credentials. If you don't know it, contact your IT helpdesk.
